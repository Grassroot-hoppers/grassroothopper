THE TOOLS

Two tools for Chez Julien.

Real retail problems, solved with open-source code, documented as playbooks for any shop to replicate.

Tool 01

Sales Prediction Engine

Three years of POS data meets weather forecast correlation. Sunny day? More mozzarella. Rainy Tuesday? Blue cheese and comfort food. The tool big retail chains have — built open-source for the corner shop.

PythonWeather APIPOS Data

Input: 3 years of historical sales data + 7-day weather forecast

Output: Daily demand predictions by product category, ordering suggestions

Tool 02

Interactive Workflow Map

An HTML-based visual map of every workflow in the shop — opening, closing, receiving deliveries, preparing orders, customer service routines. Click a station, see the checklist. New employee? Hand them this.

HTMLInteractiveVisual

Input: Shop floor layout + operational procedures

Output: Clickable floor map with step-by-step workflows per station

DEV LOG

Daily progress. No fluff.

Every day's decisions, breakthroughs, and dead ends — logged as they happen. This is what building in the open actually looks like.

Day 1 Day 5
Monday, March 10 — Day 1
Setup & Data Archaeology

Export and clean 3 years of POS data. Set up the project repo. Connect weather API. Map the shop floor for the workflow tool. Define the data schema.

Dev log entries will appear here during the hackathon.

Tuesday, March 11 — Day 2
Correlation Engine

Build the weather-sales correlation model. Which products move on sunny days? Which on rainy? Find the patterns in 3 years of data. First prediction accuracy tests.

Wednesday, March 12 — Day 3
Dashboard & Workflow Draft

Build the prediction dashboard — visual, browser-first, no install. Start the workflow map: floor layout, station definitions, checklist structure. First interactive prototype.

Thursday, March 13 — Day 4
Integration & Polish

Connect the 7-day forecast to the prediction engine. Polish the workflow map — add every procedure, test with real shop scenarios. Refine the UI for both tools.

Friday, March 14 — Day 5
Ship & Document

Deploy both tools. Write the playbook: every decision, every data source, every architecture choice. Package as a replicable template. Ship it. Done.

WHY THIS MATTERS

This is Build → Document → Replicate in action.

Big retail chains have demand prediction, staff workflow systems, inventory optimization. Small shops don't — because the tools were never built for them.

This hackathon builds two of those tools, documents every step, and publishes everything so the next shop owner doesn't start from zero. That's the Grassroots Hoppers method, applied to a real problem, in real time.

Both tools will be released under AGPL-3.0 on GitHub.
