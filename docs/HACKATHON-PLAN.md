# David Toolkit Hackathon Plan

Dates: March 9 to March 13, 2026

This sprint has three tracks. The point is not only to ship a demo. The point is to ship a demo that reads like a serious open-source project.

## Track A — Product Demo

Ship David Toolkit as a separate contributor-ready repository.

- Repo: [Grassroot-hoppers/david-toolkit](https://github.com/Grassroot-hoppers/david-toolkit)
- Shape: browser-first intelligence center for small shop owners
- Wedge: old POS exports in, actionable dashboard out
- Scope:
  - 2024-2025 item intelligence
  - 3-year macro context where finance data exists
  - curated correction layer for top SKUs
  - weather and holiday overlays
  - explicit raw-vs-interpreted evidence

## Track B — Workflow Map

Keep the Chez Julien Workflow Map in this umbrella repo as a strategic companion artifact.

- Audience: contributors, press, and future operators
- Positioning: ideal future-state operating model, not a mirror of current Notion usage
- Required workflows:
  - opening
  - morning ordering
  - receiving deliveries
  - shelf and freshness control
  - closing and cash
  - weekly supplier review
  - staffing and admin cycle

## Track C — Open-Source Rigor + Public Reporting

Use the hackathon to establish legitimacy, not just momentum.

- Code license for David Toolkit: `AGPL-3.0-only`
- Docs/content license for this repo: `CC BY-SA 4.0`
- Daily reporting:
  - one short note per day
  - one screenshot or GIF per day
  - one final retrospective
- Publishing surfaces:
  - long-form: `julien.care`
  - canonical code home: GitHub
  - social amplification: Fosstodon

## Definition Of Success

- An outsider can run the product demo locally in under 10 minutes.
- The product repo looks credible to an open-source contributor.
- The workflow map clearly shows what humans do, where software helps, and what future autonomy means.
- The sprint leaves behind reusable public artifacts: docs, screenshots, issues, and a retrospective.
