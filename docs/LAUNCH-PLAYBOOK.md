# Launch Playbook

This is the release order for the March 9 to March 13, 2026 hackathon.

## Before Posting Anywhere

- David Toolkit repo is public and runnable
- README has a screenshot, setup steps, and a clear scope statement
- at least 3 starter issues exist, including one `good first issue`
- workflow map page is live on `grassroothopper.com`
- screenshots or GIFs exist for the daily devlog

## Daily Rhythm

1. Ship the artifact first.
2. Capture one screenshot or GIF.
3. Publish the short devlog note on GitHub and expand it on `julien.care` only if the day produced a real insight.
4. Post the shortest useful summary on Fosstodon with a direct link.

## Show HN Rule

Post to Show HN only if all of this is true:

- the demo is runnable without signup
- the repository is public
- the README explains what to try in under a minute
- the product is usable enough that strangers can form an opinion without talking to Julien

If that bar is not met, publish the retrospective first and do a normal Hacker News submission later.

## Opensource.com Rule

Pitch only after the repo hygiene work is genuinely finished.

That means:

- clear licensing
- contributor docs
- a coherent public story
- proof that the project exists beyond a manifesto

## Core Story

The first story is not “we launched a movement.”

The first story is:

> I built my first serious open-source retail tool in public, using the messy reality of an old shop POS as the starting point.
